# 实验配置与参数说明

本文档说明当前统一实验框架中如何切换数据集、实验模式、优化方法和参数。所有配置统一写在 [src/ExperimentConfig.h](/d:/CVProject/AAMED_OpenCV/src/ExperimentConfig.h:1)。

## 1. 运行模式

通过以下宏切换：

```cpp
#define EXP_RUN_MODE_SINGLE_IMAGE 1
#define EXP_RUN_MODE_DATASET_BATCH 2
#define EXP_RUN_MODE EXP_RUN_MODE_DATASET_BATCH
```

- `EXP_RUN_MODE_SINGLE_IMAGE`：单张图调试，适合看 `demo/` 下 badcase 的可视化结果。
- `EXP_RUN_MODE_DATASET_BATCH`：整套数据集批量实验与评测。

## 2. 数据集选择

通过以下宏切换：

```cpp
#define EXP_DATASET_PRESET_PRASAD 1
#define EXP_DATASET_PRESET_CONCENTRIC_SYNTHETIC 2
#define EXP_DATASET_PRESET_CONCURRENT_SYNTHETIC 3
#define EXP_DATASET_PRESET_RANDOM_1 4
#define EXP_DATASET_PRESET_SMARTPHONE_2 5
#define EXP_DATASET_PRESET EXP_DATASET_PRESET_PRASAD
```

当前已接入的五套数据集：

- `Prasad Images - Dataset Prasad`
- `Concentric Ellipses - Dataset Synthetic`
- `Concurrent Ellipses - Dataset Synthetic`
- `Random Images - Dataset #1`
- `Smartphone Images - Dataset #2`

对应的数据集约定如下：

- `Prasad`
  - 路径：`dataset/Prasad Images - Dataset Prasad`
  - GT 命名：`gt_图片名.txt`
  - GT 约定：`XYRad`
  - overlap 阈值：`0.8`

- `Concentric Synthetic`
  - 路径：`dataset/Concentric Ellipses - Dataset Synthetic`
  - GT 命名：`图片文件名 + .txt`
  - GT 约定：`ConcentricDeg`
  - overlap 阈值：`0.95`

- `Concurrent Synthetic`
  - 路径：`dataset/Concurrent Ellipses - Dataset Synthetic`
  - GT 命名：`图片文件名 + .txt`
  - GT 约定：`ConcentricDeg`
  - overlap 阈值：`0.95`

- `Random #1`
  - 路径：`dataset/Random Images - Dataset #1`
  - GT 命名：`gt_图片名.txt`
  - GT 约定：`XYDeg`
  - overlap 阈值：`0.8`

- `Smartphone #2`
  - 路径：`dataset/Smartphone Images - Dataset #2`
  - GT 命名：`gt_图片名.txt`
  - GT 约定：`XYDeg`
  - overlap 阈值：`0.8`

补充说明：

- 如果数据集目录下存在 `imagenames.txt`，框架会优先按它读取图像列表。
- 如果没有 `imagenames.txt`，框架会自动遍历 `images/` 目录并按文件名排序。

## 3. 方法开关

当前统一框架支持的方法：

```cpp
#define EXP_ENABLE_WEIGHTED_ARC 0
#define EXP_ENABLE_MULTI_SCALE_FPN 0
#define EXP_ENABLE_SMALL_ELLIPSE_GUARD 1
#define EXP_ENABLE_RESERVED_METHOD_THIRD 0
```

含义：

- `EXP_ENABLE_WEIGHTED_ARC`：开启加权弧图 / 软兼容 AAM。
- `EXP_ENABLE_MULTI_SCALE_FPN`：开启多尺度上采样 + 跨尺度严格共识融合。
- `EXP_ENABLE_SMALL_ELLIPSE_GUARD`：开启小椭圆验证守门模块，用于抑制小尺寸伪椭圆。
- `EXP_ENABLE_RESERVED_METHOD_THIRD`：为未来第三种方法预留，当前未实现。

## 4. 已接入方法与参数

### 4.1 weighted-arc

参数结构体：`WeightedArcConfig`

```cpp
struct WeightedArcConfig
{
    bool enable = EXP_ENABLE_WEIGHTED_ARC != 0;
    float softLinkThreshold = 0.58f;
    int maxSoftNeighborsPerDirection = 8;
    double pairCompatGain = 0.18;
    double crossCompatGain = 0.24;
    double singleCompatGain = 0.22;
    bool selectBestValidatedCandidate = true;
};
```

说明：

- `softLinkThreshold`：软连接生效阈值。
- `maxSoftNeighborsPerDirection`：每个方向最多补多少个 soft 邻居。
- `pairCompatGain / crossCompatGain / singleCompatGain`：组合排序时的兼容度增益。
- `selectBestValidatedCandidate`：是否从所有通过验证的候选中选择最优，而不是遇到第一个就停止。

备注：这组参数来自此前 `method-weighted-arc` 分支的正优化结果，是当前该方法的已知最优配置。

### 4.2 multi_scale_fpn

参数结构体：`MultiScaleFpnConfig`

```cpp
struct MultiScaleFpnConfig
{
    bool enable = EXP_ENABLE_MULTI_SCALE_FPN != 0;
    std::vector<int> scales = {2, 3};
    std::vector<double> validationThresholds = {0.77, 0.72};
    double fusionIoU = 0.8;
    int minBranches = 3;
    bool requireCrossScale = true;
    double remapRadiusMax = 20.0;
    bool useCubicInterpolation = true;
};
```

说明：

- `scales = {2, 3}`：对原图做 2 倍、3 倍上采样。
- `validationThresholds = {0.77, 0.72}`：分别使用严格阈值和宽松阈值运行 AAMED。
- `fusionIoU = 0.8`：跨分支融合时判定为同一椭圆的 overlap 门槛。
- `minBranches = 3`：至少被几个分支同时支持。
- `requireCrossScale = true`：必须同时得到 `2x` 与 `3x` 支持。
- `remapRadiusMax = 20.0`：只补充回映射后的小椭圆。
- `useCubicInterpolation = true`：上采样采用三次插值。

备注：这组参数来自 `multi_scale_fpn` 文件夹中的最佳配置验证结果，是当前该方法的已知最优配置。

### 4.3 small_ellipse_guard

参数结构体：`SmallEllipseGuardConfig`

```cpp
struct SmallEllipseGuardConfig
{
    bool enable = EXP_ENABLE_SMALL_ELLIPSE_GUARD != 0;
    float minAxisThreshold = 9.0f;
    float missingEdgeCompensation = 0.40f;
    float strongGradientScoreThreshold = 0.72f;
    float minEdgeCoverage = 0.30f;
    float minStrongGradientCoverage = 0.45f;
    bool hardRejectWeakSmallCandidates = true;
};
```

说明：

- `minAxisThreshold`：仅对短轴小于该值的小椭圆生效。
- `missingEdgeCompensation`：无边缘采样点时的小椭圆补偿分。
- `strongGradientScoreThreshold`：单个采样点被视为强梯度支持点的阈值。
- `minEdgeCoverage`：小椭圆最少边缘覆盖率。
- `minStrongGradientCoverage`：小椭圆最少强梯度覆盖率。
- `hardRejectWeakSmallCandidates`：是否直接剔除弱边缘、弱梯度的小候选。

备注：这组参数直接沿用 `ljz/Validation.cpp` 与 `ljz/优化.md` 中给出的配置。当前只是完成统一框架接入，还没有像 `weighted-arc` 和 `multi_scale_fpn` 那样完成系统调参验证。

## 6. 输出目录命名规则

输出目录根据当前方法组合自动命名，例如：

- `output/baseline`
- `output/weighted_arc`
- `output/multi_scale_fpn`
- `output/small_ellipse_guard`
- `output/weighted_arc__multi_scale_fpn`
- `output/weighted_arc__multi_scale_fpn__small_ellipse_guard`

## 7. 当前参数是否已经是“最优”

- `weighted-arc`：是，当前参数就是此前独立分支验证得到的正优化配置。
- `multi_scale_fpn`：是，当前参数就是同学在 `multi_scale_fpn` 中验证得到的最佳配置。
- `small_ellipse_guard`：不是，目前只是按 `ljz` 给出的默认方案完成集成，还没有在统一框架里完成系统调参与最优组合搜索。

## 8. 推荐实验顺序

1. 先分别单开三种方法，验证是否与各自独立实现结果一致。
2. 再测试两两组合：
   - `weighted_arc + multi_scale_fpn`
   - `multi_scale_fpn + small_ellipse_guard`
   - `weighted_arc + small_ellipse_guard`
3. 最后测试三者联用。
4. 每组实验统一记录：
   - `Precision`
   - `Recall`
   - `FMeasure`
   - `AverageDetectedTimeMs`
