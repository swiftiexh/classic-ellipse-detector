# 实验总结

生成时间：2026-05-30

## 结论先说

当前真正推荐保留、并且属于主线有效改动的是：

```text
src/Group.cpp 的 isCombValid 修复
```

仓库里仍然保留了第一次和第二次的实验脚本、日志和文档，但它们不是都被合成为一个最终算法。当前最终有效结论是：**保留 Group.cpp 修复；GeoFilter 作为参考实验保留；SCC-Enhance 不建议继续。**

## 第一次修改：SCC-GeoFilter

第一次加的是 `SCC-GeoFilter`，它是一个独立后处理模块。

新增内容主要是：

```text
scc_experiments/scc_geofilter.py
scc_experiments/run_scc_geofilter_prasad.py
scc_experiments/run_scc_geofilter_sanity.py
```

它做的事情是读取已有的 AAMED 检测结果：

```text
datasets/prasad/AAMED/*.fled.txt
```

然后用几何规则删除少量明显不合理的椭圆候选，例如轴比异常、面积异常、中心越界、外接框越界等。

注意：第一次修改 **没有改 C++ detector 主流程**，也没有修改 `src/Group.cpp`。它只是对已有 `.fled.txt` 结果做后处理。

结果如下：

| 方法 | Precision | Recall | FMeasure | AverageDetectedTimeMs |
| --- | ---: | ---: | ---: | ---: |
| Official AAMED baseline | 0.771285 | 0.396567 | 0.523810 | 5.146824 |
| SCC-GeoFilter strict | 0.776094 | 0.395708 | 0.524161 | 5.146824 |

相对 baseline：

| 指标 | 变化 |
| --- | ---: |
| Precision | +0.004809 |
| Recall | -0.000859 |
| FMeasure | +0.000351 |

第一次结论：GeoFilter 有一点点提升，主要提升 Precision，但 Recall 略降，FMeasure 提升非常小。它可以作为参考方向保留，但不适合作为最终主线继续加 hard filter。后续更适合改成 soft score / re-ranking。

## 第二次修改：Group 修复 + SCC-Enhance

第二次包含两部分。

第一部分是核心有效修改：修复 `src/Group.cpp` 中 `BiDirectionVerification` 的 `isCombValid` 状态污染问题。

核心变化是：

```text
将 isCombValid 从 fitComb 循环外移动到每个候选循环内部初始化。
```

修复原因：

原始代码里，`isCombValid` 如果在某个候选组合中被置为 `false`，后续候选可能继承这个错误状态，导致本来应该继续验证的组合被错误跳过。修复后，每个候选组合都有独立的 `isCombValid` 状态。

第二部分是实验性修改：`SCC-Enhance`。

新增内容主要是：

```text
scc_experiments/scc_enhance.py
scc_experiments/run_fixed_baseline.py
scc_experiments/run_scc_enhance.py
scc_experiments/run_scc_enhance_sanity.py
```

它尝试在 detector 前面加 CLAHE + bilateral filter 图像预处理，希望提升输入图像质量。

第二次结果如下：

| 方法 | Precision | Recall | FMeasure | AverageDetectedTimeMs |
| --- | ---: | ---: | ---: | ---: |
| Official AAMED baseline | 0.771285 | 0.396567 | 0.523810 | 5.146824 |
| Fixed detector baseline | 0.779070 | 0.402575 | 0.530843 | 14.952466 |
| SCC-Enhance light | 0.728296 | 0.388841 | 0.506995 | 15.036036 |
| SCC-Enhance medium | 0.727119 | 0.368240 | 0.488889 | 17.242711 |
| SCC-Enhance strong | 0.783985 | 0.361373 | 0.494712 | 15.353225 |

其中 `Fixed detector baseline` 就是只保留 `Group.cpp` 修复后重新运行 detector 的结果。

相对 official baseline，Group 修复带来的变化是：

| 指标 | 变化 |
| --- | ---: |
| Precision | +0.007785 |
| Recall | +0.006008 |
| FMeasure | +0.007033 |

第二次结论：`Group.cpp` 修复是当前最有价值的改动。`SCC-Enhance` 三组预处理都没有超过 fixed detector baseline，因此不建议把当前 CLAHE + bilateral 预处理作为最终方案。

## 现在到底是什么状态

当前仓库里保留了两轮实验的文件：

```text
scc_experiments/scc_geofilter.py
scc_experiments/scc_enhance.py
scc_doc/
```

但是当前真正推荐作为最终有效改动的是：

```text
src/Group.cpp
```

更准确地说：

| 内容 | 当前状态 |
| --- | --- |
| Group.cpp isCombValid 修复 | 保留，当前主线有效改动 |
| SCC-GeoFilter | 保留为第一次实验结果，未接入 detector 主流程 |
| SCC-Enhance | 保留为第二次实验记录，但结果不好，不推荐继续 |
| 当前最终算法 | 不是“第一次 + 第二次”的组合，而是以 Group.cpp 修复为主 |

## 推荐下一步

推荐下一轮不要继续扩大 hard filter，也不要继续当前 hard-coded SCC-Enhance。

更建议做：

```text
SCC-GeoScore：候选级 soft geometry scoring / confidence calibration
```

也就是把第一次 GeoFilter 中的几何先验，从“硬删除候选”改成“软评分或重排序”，并建立在第二次 `Group.cpp` 修复后的 detector 输出之上。
