
# 改进 classic-ellipse-detector 算法的实验任务

请在当前项目 `E:\scc_desk\Net\classic-ellipse-detector` 中完成一次以“提高椭圆检测精度或速度”为目标的算法迭代。
当前 git 分支为 `scc`，后续需要合并，因此请优先采用**新增文件、新增模块、新增实验脚本**的方式实现，尽量不要直接修改已有核心文件。如果确实必须修改已有文件，请说明原因，并保持改动最小。

## 一、任务目标

本次任务不是单纯调参，而是寻找一个适合从传统算法迁移到 CNN / 深度学习中的模块化改进思路。

请重点考虑以下类型的改进：

1. 能在传统算法中独立验证有效；
2. 后续可以迁移为 CNN 中的模块、loss、后处理、候选筛选策略或几何约束；
3. 对精度或速度至少有一个方向产生可测量影响；
4. 工程实现尽量轻量，方便后续复现实验和合并代码。

## 二、前期阅读与调研

请先阅读项目代码和实验流程，明确：

1. 当前算法整体流程；
2. 数据输入和输出格式；
3. baseline 的运行命令；
4. 评估脚本和评价指标；
5. 当前耗时主要出现在什么阶段；
6. 当前误检、漏检可能出现在哪些环节。

然后调研 2022–2026 年与以下方向相关的研究论文：

1. ellipse detection；
2. edge grouping；
3. geometric primitive detection；
4. line / arc / ellipse proposal；
5. CNN-based ellipse detection；
6. differentiable geometry constraint；
7. lightweight post-processing。

每篇论文只需要提取和本项目有关的内容，重点回答：

* 这篇论文提出了什么可迁移模块？
* 能否先在传统算法中验证？
* 预计提高精度还是速度？
* 实现成本高不高？
* 是否适合以后迁移到 CNN？

## 三、候选 idea 选择

请至少提出 3 个候选 idea，并从下面几个角度打分：

1. 预期精度提升；
